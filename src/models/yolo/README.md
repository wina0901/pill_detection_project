# YOLO 경구약제 객체 탐지 실험 기록 (v1(=p1) ~ p4)

## 1) 프로젝트 목표
- 73개 경구약제 클래스를 객체검출로 탐지
- 최종 제출 성능(캐글 Public Score) 최대화
- 모델 성능 + 운영 속도(추론/학습 시간) 균형 확보

## 2) 핵심 정정 사항 (중요)
- `v2`는 **무튜닝 베이스라인**이 맞습니다.
- 다만 `v2`를 “무전처리”로 보면 안 됩니다.
  - 실제 학습은 `data.yaml`의 `letterbox_images/train`, `letterbox_images/val`을 사용했습니다.
  - 즉, **추가 튜닝/증강은 없었지만 letterbox 전처리 데이터는 사용**했습니다.

## 2) 실험 타임라인 요약

### v1~v3: 모델 스크리닝 및 실험 기준 설계(YOLO8n/8s/26s/26m)
- 목적: 초기 베이스라인 및 모델군별 속도/성능 감 파악
- 결과: `8s`, `11m` 계열이 실제 운영/성능 균형 측면에서 유효
- 비고: 26m는 학습비용이 커서 실험 효율 저하
- 기준선 확정, LR/파인튜닝 , 방향 설정

### p2_v1: 본격 비교 (전처리 + LR 튜닝 + CP A/B + 단계적 파인튜닝 + 앙상블)
- 이미지 전처리 이슈로 실험 새로 시작
- 구성:
  - 무튜닝
  - LR 튜닝
  - CP 적용/미적용 A/B
  - Freeze→Unfreeze 파인튜닝
  - 단일/앙상블 제출 비교
- 핵심 제출 결과:
  - `submission_yolo11m_0318_n.csv` → **0.99073** (무튜닝)
  - `submission_yolo11m_cp_ft_uf.csv` → **0.99036** (단일 최고점)
  - `submission_yolov8s_no_cp_ft_ext10.csv` → **0.98565**
  - `submission_ens_v8s_ext10__11m_uf.csv` → **0.99093 (캐글 최고점)**
- 결론: 단일 최고는 11m, 최종 최고점은 8s+11m 앙상블
- 추가 인사이트: 과제 난이도 특성상 강한 튜닝보다 기본 파이프라인이 매우 강함

### p2_v2: YOLO 내장 증강 도입 실험
- 목적: 내장 증강으로 추가 개선 가능성 확인
- 결과:
  - `submission_ens_v8s_11m_augon.csv` → **0.96248**
- 결론: 내장 증강이 유의미한 개선으로 연결되지 않음(오히려 하락)

### p3: 소수 클래스 보강 1차
- 목적: 저성능 클래스 AP 개선
- 관찰:
  - 일부 소수 클래스 AP 개선
  - 반대로 기존 안정 클래스 성능 하락(트레이드오프)
- 결론: 소수클래스 보강은 “전체 mAP 자동 상승”이 아니라 클래스 간 균형 문제로 접근 필요

### p4: 블랙 마스킹 + 추가 실험
- 목적: 혼동 클래스(외형 유사 클래스) 오분류 완화
- 제출 결과(해상도 800):
  - `submission_ens_v8s_mask_11m_mask_tta.csv` → **0.98455**
- 고해상도(1024) 앙상블:
  - `submission_ens_1024_v8s_mask_11m_mask_tta.csv` → **0.95788**
- 결론: 블랙마스킹/해상도 증가는 본 실험에서는 점수 갱신 실패

## 3) 핵심 결과 표 (최종 비교)

| 구분 | 대표 Run / CSV | VAL mAP50-95 (기록 기준) | Kaggle Public Score | recall | precision |
|---|---|---:|---:|---:|---:|
| 11m 싱글(무튜닝, v2) | `submission_yolo11m_0318_n.csv` | 0.891920 | **0.99073** | 0.941995 | 0.810379 |
| 8s+11m 앙상블(최고점) | `submission_ens_v8s_ext10__11m_uf.csv` | 0.902814 | **0.99093** | 0.989559 | 0.808531 |
| 8s 싱글(최고 mAP run 기준) | `submission_yolov8s_no_cp_ft_ext10.csv` | 0.913707 | **0.98565** | 0.953596 | 0.813861 |
| 블랙마스킹 앙상블(800) | `submission_ens_v8s_mask_11m_mask_tta.csv` | 0.894225 | **0.98455** | 0.960000 | 0.762500 |


> 참고: VAL 지표는 실험 시점/평가 함수(common vs native) 기준 차이가 있으므로, 최종 비교 시 동일 기준으로 재정렬 필요.


## 4) 운영 관점 요약

- **정확도 최우선 제출 후보**
  - 11m 단일, 8s+11m 앙상블
- **속도/경량 우선 후보**
  - 8s 계열 (학습/운영 비용 유리)
- **실무 결론**
  - 본 데이터셋에서는 “복잡한 증강/전처리”보다
  - **강한 베이스라인 + 안정적 파인튜닝 + 보수적 앙상블**이 가장 재현성 높은 전략

## 5) 현재 결론
- 최종적으로 제출 성능은 **11m 싱글 + 8s/11m 앙상블** 조합이 가장 안정적이었습니다.
